# Tower Defense

TypeScript (Bun) tower defense game with headless simulation for balance tuning.

## Running the game

**Requirements:** Docker (or Bun locally in `player/`).

```bash
make dev
```

Open [http://localhost:5173](http://localhost:5173). The player auto-fires at enemies moving toward the center; kills grant gold to buy upgrades (damage, life, regen, attack speed).

## Commands (project root)

| Command | Description |
|---------|-------------|
| `make dev` | Dev server with hot reload (port 5173) |
| `make build` | Production build → `player/dist/` |
| `make preview` | Serve the build (after `make build`) |
| `make simulate` | Headless simulation: 3 profiles (full, two, one), stats and reports in `player/rapport/` |
| `make ci` | Install deps + typecheck, tests, lint |
| `make shell` | Shell in container (Bun) |

Simulation examples:

```bash
make simulate                    # 100 runs per profile (default)
make simulate RUNS=50            # 50 runs per profile
make simulate PARAMS=other.json  # Different params file
```

## Structure

- **`player/`** — Main app (source, build, params).
  - `player/src/` — TypeScript code (domain, infrastructure).
  - `player/gameparams.json` — Game parameters (balance); edit after analysing reports.
  - `player/rapport/` — Simulation reports (generated by `make simulate`).
- **`docs/`** — Documentation (ROADMAP, balance analysis prompt, etc.).

## Balance

Goal: **normal curve** with **peak at ~8 min** survival, **max 10 min** per run. The three simulation profiles (full = 4 stats, two = 2 stats, one = 1 stat) should have a mean close to 8 min.

1. Run `make simulate` (e.g. `RUNS=100`).
2. Check the latest report in `player/rapport/` (fields `pctOfPeak`, `meanFrames`, etc.).
3. Adjust `player/gameparams.json` from the numbers.
4. See `docs/PROMPT_ANALYSE_EQUILIBRAGE.md` for a sample analysis and tuning prompt.

## Development without Docker

From `player/`:

```bash
cd player
bun install
bun run dev      # or build, test, typecheck, lint
bun run src/simulate.ts 100 . gameparams.json
```
